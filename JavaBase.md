##1.计算机系统机制概述
- 数制是一组固定的符号和统一的额规则来表示数值的方法
- 计算机底层使用的数制是二进制
- 用Java编程使用的是十进制。Java底层仍使用二进制
- 计算机常用的数制还有八进制和十六进制

### 1.1十进制
>十进制的基本数字是0~9 ，逢十进位。
>10称作“基数”，10^n（10的n次幂）被称作“权”

----
### 1.2 二进制
>二进制的基本数字是0、1，逢2进位。
>二进制的基数为2，权为2^n（2的n次方）
>（1111）二进制转为十进制    --->（15）

### 1.3 十六进制
>十六进制是二进制的简写，方便专业人员书写二进制数据；十六进制的基数为16，权为16^n （16的n次方）；
>再Java代码中十六进制数用0x或0x做前缀（0是数字零，而不是字母O）
>0x1000=1*16^3     0x1000=1*16^3


### 1.4 十六进制与其他进制的转换
>41的十六进制   等价于  65 的十进制
>16进制的1位数代表2进制的4位数。
>41的十六进制等价于    0100 0001 的二进制

### 1.5 二进制转换为十进制/十六进制
```
二进制转为十进制
   (10110)2 = (22)10

二进制转为十六进制
	(11010110)2= (D6)16
```
>规律：二进制转为十六进制数相当于一位十六进制数。

>计算机最基本的存储单位是字节（byte），一个字节有8位。8位2进制的最大值=（11111111）2	

### 1.6 十进制数转化为二进制数/十六进制数
>转二进制数规律：不断除以2，保留余数，商为0时，不在除2，将所有余数倒序排序
>（13）10  = （1101）2

----
>转十六进制数规律：不断除以16，保留余数，商为0时不再除16，将所有余数倒序排序。
>（138）10 = （8A）16

### 2.8 二进制补码
```
计算机系统的内部以二进制形式存储数据。
在Java程序中输入的十进制的数据都会被自动转换为二进制，Java内部也以二进制数来进行数值运算，但返回的结果是十进制。
```
- 补码规则
```
在计算机系统中，数值一律用二进制的补码来存储。
二进制的最高为是符合位，0表示正数，1表示负数

正数的值是其本身，负数的值是最高位不变，其他逐位取反，再加1
```
>（1101）2     ==》取反 0010   再加1   0011    符号位不变   -3

>两数相加，若最高位（符号位）有进位，则进位被舍弃。


- 为什么使用补码
```
1.可以将符号位和其他位统一处理；
2.最高为不再表示数值，二十作为符号位，正好将数值折半，即一般是0至正数，一半是负数。
3.减法也可以按加法来处理；
```

- 补码运算的特征
```
计算机中正数和负数的关系是取反加一
```

- 补码运算的原理
```
正数+负数=模
  
  模：某种类型数据的总数
```

### 3.0 浮点类型
```
浮点类型用于表示小数的数据类型。
浮点数原理：也就是二进制数科学记数法。
Java的浮点类型有float金额double两种。
Java默认浮点类型计算的结果是double类型，字面量也是double类型。
    若要给float类型的变量赋字面量的值，则要在字面量后面加F(f)
```
>浮点运算就是实数运算，由于计算机只能存储整数，所以实数都是约数，这样浮点运算是很慢的而且还会有误差。
